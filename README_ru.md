# mewgenics-autobattle

**mewgenics-autobattle** — это мод для игры Mewgenics, который значительно улучшает искусственный интеллект (ИИ) кошек во время авто-боев. Вместо того чтобы полагаться на случайность или базовую логику, этот мод использует специализированные предустановки (пресеты) для каждого класса, обеспечивая умное и эффективное поведение в бою.

Танки будут блокировать урон и защищать союзников, лекари — отдавать приоритет спасению умирающих друзей, а не нанесению незначительного урона врагу. Убийцы (вор/thief) будут искать возможность удара в спину, а Некроманты — эффективно использовать трупы, избегая суицидальных заклинаний.

## Как использовать
1. Установите мод, поместив его в папку модов игры Mewgenics.
2. Мод автоматически активируется во время любого авто-боя.
3. Ваши кошки сразу начнут использовать новые профили поведения (`smart_fighter`, `smart_mage`, `smart_cleric` и т.д.).

## Релизный процесс
- Единый источник версии: `VERSION`.
- `description.json` и `CHANGELOG.md` обновляются через `scripts/version.sh`.
- Повысить версию:
  - `scripts/version.sh bump patch --note "Краткая заметка релиза"`
  - `scripts/version.sh bump minor --note "Краткая заметка релиза"`
  - `scripts/version.sh bump major --note "Краткая заметка релиза"`
- Собрать архив `mewgenics-autobattle.<ext>`:
  - `scripts/release.sh zip`
  - `scripts/release.sh tar`
  - `scripts/release.sh 7z`
  - `scripts/release.sh rar`
  - `scripts/release.sh all`
- Папка выхода: `dist/`.
- Каждый архив содержит только `data/` и `description.json`.
- Автоматизация через GitHub:
  - Запушьте тег `vX.Y.Z` (например `v1.2.0`) и workflow создаст/обновит GitHub Release.
  - Текст релиза берется из соответствующей секции `CHANGELOG.md` (`## [X.Y.Z]`).
  - Архивы из `dist/` прикладываются как assets к релизу.

## Как модифицировать и помогать в разработке
Вы можете настраивать существующее поведение ИИ или добавлять свое, изменяя файлы внутри директории `data/`. Когда мод загружается, игра применяет файлы из папки `data/` поверх оригинальных файлов игры, используя специальную систему патчинга Mewgenics.

### Формат GON и Механика Патчинга
В Mewgenics используется формат конфигурационных файлов **GON** (Glaiel Object Notation). Это легковесный формат, в котором данные разделяются пробелами.
Вместо того чтобы целиком заменять оригинальные файлы, мод использует специальные расширения, чтобы указывать игре, как объединять ваши изменения:
- `[имя_файла].gon.patch`: Изменяет целевой файл, используя внутренние команды. Внутри `.patch` файла вы можете добавлять такие модификаторы, как `.overwrite`, `.append`, `.merge`, `.add` и `.multiply` к отдельным полям или массивам.
- `[имя_файла].gon.merge`: Выполняет глубокое слияние файла. Он находит указанные блоки и изменяет/добавляет исключительно эти поля, не трогая остальной файл.
- `[имя_файла].txt.append`: Склеивает два файла вместе (например, добавление в списки).
- `[имя_файла].csv.merge`: Перезаписывает значения в CSV, но требует сохранения пустых ячеек с запятыми `,,,` для неизмененных данных (что не всегда удобно при патчах игры).

*Пример операции `.merge`:*
```json
smart_tank.merge {
    damage_ally -9999
    damage_enemy 1
    heal_ally 1
}
```

### Основные компоненты ИИ
Изменения ИИ разбиты на три основные категории внутри папки `data/`:

#### 1. Пресеты решений / Decision Presets (`data/ai_presets/decision_presets.gon.patch`)
Этот файл присваивает очкам (Scores) определенную ценность. ИИ оценивает все доступные действия и выбирает то, которое наберет больше всего очков.
- **Веса (Weights):** Вы можете задавать приоритет таким действиям, как `damage_enemy`, `heal_ally`, `buff_self`, `kill_enemy`, `spawn_object` и другим.
- **Безопасность:** Параметры вроде `damage_ally -9999` гарантируют, что ИИ случайно не убьет своих.
- **Правила:** Флаги вроде `consider_aoe`, `consider_overkill` и `accurate_knockback` меняют комплексную логическую оценку.

#### 2. Пресеты передвижения / Movement Presets (`data/ai_presets/move_presets.gon.patch`)
Определяет, на какие клетки хотят вставать кошки.
- **Дистанции:** Настройте `distance_to_enemy`, `distance_to_ally`, `distance_to_corpse` и идеальную дистанцию `preferred_distance` (которая поддерживает вычисления вроде `mov+reach`). Позитивные числа означают "держаться подальше", негативные — "подойти вплотную".
- **Выживаемость:** `danger_avoidance` определяет, насколько ИИ боится ловушек (шипы, лава). `tall_grass` заставляет юнитов прятаться в кустах.

#### 3. Патчи способностей (`data/abilities/*.gon.patch`)
Вы можете внедрять ситуативную логику напрямую в конкретные способности, используя **Условные блоки (Conditionals)**. Самый популярный метод — выдача фиксированного бонуса `FlatAIBonus`, если выполняется условие `Conditional_Speculative`.

*Пример: Давать высокий приоритет способности ТОЛЬКО если она бьет в спину (Backstab):*
```json
SomeAttack.merge {
    damage_instance {
        ai_base_score 10
        effects {
            Conditional_Speculative {
                Conditional_Backstab {}
                Then { FlatAIBonus 50 }
            }
        }
    }
}
```
В базовой игре заложены десятки таких условий: `Conditional_Ally`, `Conditional_Enemy`, `Conditional_TargetHasStatus`, `Conditional_HealthThreshold`, `Conditional_Boss` и другие. Вы можете комбинировать их, создавая невероятно сложное и умное поведение на поле боя!

### Файлы базовой игры, которые можно менять
В оригинальной игре Mewgenics есть множество конфигурационных файлов и папок. Вы можете менять их, создавая `.patch` или `.merge` файлы в своей папке `data/`, что позволяет изменить практически любой аспект игры:

#### Искусственный интеллект (`ai_presets/`, `abilities/`)
- **`decision_presets.gon`**: Изменение базовых профилей ИИ (careless, angry) для переработки "характера" классов.
- **`move_presets.gon`**: Тотальный контроль позиционирования: отношения к ловушкам (`danger_avoidance`), воде (`distance_to_water`) и укрытиям.
- **`abilities/`**: Настройка точных формул урона, форм радиуса атаки (AOE), вероятности прока статусов и логики ИИ для абсолютно каждого навыка.

#### Кошки и Классы (`characters/`, `classes/`, `catgen.gon`)
- Изменение базовых характеристик, стартовой экипировки и кривой прокачки разных классов.
- Настройка параметров генерации кошек, расширение словаря их имен (`catnames_*.txt`) и боевых реплик (`catquotes.gon`).
- Редактирование мутаций, травм и пассивных навыков (`mutations/`, `passives/`, `injuries.gon`).

#### Предметы и Экипировка (`items/`, `item_pools/`)
- Ребаланс оружия, брони и расходников.
- Изменение шансов выпадения лута из сундуков и настройка сетовых бонусов (`item_setbonuses.gon`).

#### Мир, Карта и Враги (`maps/`, `spawns.gon`, `events/`)
- **`spawns.gon` и `maps/`**: Контроль появления врагов, их количества в разных биомах и правил генерации комнат.
- **`events/` и `npc_scripts/`**: Написание собственных текстовых квестов, диалогов с NPC и случайных встреч на карте мира.
- **`shops/`**: Изменение ассортимента и цен в магазинах.

#### Механики Дома (`house.gon`, `furniture_effects.gon`)
- Изменение баффов от мебели (например, для разведения кошек) и влияния погоды на настроение в доме (`house_weather.gon`).

#### Интерфейс и Текст (`text/`, `damage_text_styles.gon`)
- Полный перевод игры на другие языки (локализация) или изменение описаний.
- Настройка отображения вылетающего урона в бою.
